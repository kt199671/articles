# Tatami Works 月間振り返り - note記事作成スキル

コワーキングスペース「Tatami Works」の月間振り返りnote記事を対話形式で作成するスキルです。

## いつこのスキルを使うか

以下のような依頼があった場合に自動的に発動してください：

- 「月間振り返り記事を作成」
- 「Tatami 振り返りを書いて」
- 「tatami retrospective」
- 「運営記録を書いて」
- 「Tatami Works の振り返り」
- 「Tatami 振り返り」

## 記事の保存先

```
/home/kentaro/src/kentaro/articles/note/magazine/Tatami運営記録/
```

ファイル名形式: `【コワーキングスペース Tatami Works】 YYYY年MM月振り返り.md`

## 作成フロー

### Phase 1: 情報収集（Interactive）

1. **既存記事の確認**
   - 同フォルダ内の `.md` ファイルをGlobで検索
   - 既に作成済みの月を特定し、重複を防ぐ
   - **欠けている月を自動検出して積極的に提案**
     - 例: 「2025年7月が未作成です。7月の記事を作成しますか？それとも別の月にしますか？」
     - 欠けている月が複数ある場合は、古い順に提案

2. **基本情報の質問（AskUserQuestionで1問ずつ）**

   **質問1: 対象月の確認**
   ```
   どの月の振り返り記事を作成しますか？

   [既存記事の状況を表示]
   - 作成済み: 2025年4月、5月、6月、8月、9月、10月、11月
   - 未作成: 2025年7月、12月

   形式: YYYY年MM月（例: 2025年7月）
   ```

3. **質問2: 絵文字の選択**
   ```
   今回使用する絵文字を選んでください:

   1. 🐨（コアラ）
   2. 🐻‍❄️（ホッキョクグマ）
   3. 🐻（クマ）

   数字で選択してください（1-3）
   ```

4. **質問3: トピック数**
   ```
   今月のメイントピックはいくつありますか？（2-4個を推奨）

   数字で入力してください:
   ```

5. **トピックごとの詳細質問（トピック数分繰り返し）**

   各トピックを丁寧に深掘りするため、3段階で質問してください：

   **質問A: トピックタイトル**
   ```
   トピック[N]のタイトルを教えてください:
   （例: Wifi高速化！、カイゼン記録作成！）
   ```

   **質問B: トピックカテゴリ**
   ```
   このトピックのカテゴリを選んでください:

   1. 運営変更（営業時間、予約制度、ルールなど）
   2. 設備改善（家具、機器、WiFiなど）
   3. サービス追加（メニュー、価格、新プランなど）
   4. イベント・コミュニティ（勉強会、交流会など）
   5. 教育プログラム（Tatami Study関連）

   数字で選択してください（1-5）:
   ```

   **質問C: トピック詳細**
   ```
   [トピックタイトル]について、以下の情報を教えてください:

   - 何を行ったか/変更したか
   - その背景や理由
   - 期待される効果や反応
   - 関連するURL（あれば）

   自由に記述してください:
   ```

6. **質問4: 数値データ（オプション）**
   ```
   利用者数やメンバー数などの数値データはありますか？

   あれば記載してください（なければスキップ）:
   - 利用者数の増減
   - 新規会員数
   - その他の統計
   ```

7. **質問5: 振り返りポイント**
   ```
   今月の振り返りとして含めたいポイントを教えてください:

   - 直面した課題
   - 学んだこと
   - 今後の改善計画
   - 感謝したいこと

   自由に記述してください:
   ```

### Phase 2: 計画（EnterPlanMode）

1. **EnterPlanModeで記事構成を設計**

   **簡潔な計画**を作成してください（振り返り記事は構造が定型的なため）。

   計画ファイルに以下を記載:
   - 対象月と絵文字
   - 目次構成（トピックのタイトル一覧）
   - トピックの配置順序

   詳細な概要や画像配置位置は省略してOKです。

2. **ユーザー確認**
   - 構成案を提示し、修正があれば反映

### Phase 3: 執筆

1. **TodoWriteで進捗管理**

   タスク例:
   - 導入部分の作成（挨拶・概要）
   - 目次の作成
   - トピック1の執筆
   - トピック2の執筆
   - （以下トピック数分）
   - 振り返りセクションの執筆（オプション）
   - まとめの作成
   - 画像プレースホルダーの挿入
   - 文字数確認

2. **各セクションの執筆**

   **導入部（必須要素）**
   ```markdown
   こんにちは！コワーキングスペース「Tatami Works」代表の冨上（とかみ）です[絵文字]

   今回は[YYYY年MM月]の振り返りを行います。

   [今月の簡単な総括: 1-2文]
   ```

   **目次（必須要素）**
   ```markdown
   ## 目次
   1. YYYY年MM月のトピック
   2. [トピック1タイトル]
   3. [トピック2タイトル]
   4. [振り返りセクション名]（オプション）
   5. まとめ
   ```

   **トピックセクション（### 見出し）**
   - カテゴリに応じた適切な表現を使用
   - 具体的な変更内容・背景・効果を記載
   - 必要に応じてURLを含める
   - 画像が必要な箇所に `<!-- 画像: [説明] -->` コメントを挿入

   **振り返りセクション（オプション）**
   - 利用者数の動向
   - 運営上の気づき
   - 課題と対応策
   - 今後の展望

   **まとめ（必須要素）**
   ```markdown
   ## まとめ

   [今月の総括: 2-3文]

   [感謝の言葉: 1-2文]

   引き続き、Tatami Worksをよろしくお願いいたします！
   ```

3. **品質チェック**
   - 文字数: 1,000-2,700文字を目標
   - 目次とセクション見出しの一致確認
   - 標準の締めくくりフレーズが含まれているか
   - 画像プレースホルダーの適切な配置

### Phase 4: 保存と報告

1. **ファイル保存**
   - 正しいパスとファイル名で保存
   - 重複確認（既存ファイルの上書き防止）

2. **完了報告**
   ```
   [YYYY年MM月]の振り返り記事を作成しました！

   保存先: /home/kentaro/src/kentaro/articles/note/magazine/Tatami運営記録/【コワーキングスペース Tatami Works】 YYYY年MM月振り返り.md

   文字数: [X]文字
   トピック数: [N]個

   次のステップ:
   - 記事を確認して必要に応じて修正
   - 画像を追加（<!-- 画像 --> コメント箇所）
   - noteに投稿
   ```

## スタイルガイド

### トーンと文体

- **パーソナル**: 代表である冨上本人の声で語る一人称視点
- **透明性**: 課題や失敗も正直に共有する
- **感謝**: 利用者への感謝を忘れない
- **コミュニティ重視**: 利用者との関係性を大切にする表現
- **謙虚**: 改善の余地を認め、学び続ける姿勢

### フォーマット規則

**見出し階層**
- `#` タイトル（使用しない - ファイル名がタイトル）
- `##` 大セクション（目次、まとめ、振り返りなど）
- `###` トピックタイトル

**絵文字使用**
- 冒頭の挨拶に1つのみ: 🐨 または 🐻‍❄️ または 🐻
- トピックタイトルにはユーザーが明示的に指定した場合のみ使用

**箇条書き**
- `-` を使用（`*` は不使用）
- ネストする場合は4スペースインデント

**強調**
- 重要な変更点や数値は `**太字**` で強調
- 過度な強調は避ける

**URL**
- リンクテキストを含む markdown 形式: `[テキスト](URL)`
- 可能な限り説明的なリンクテキストを使用

**画像プレースホルダー**
```markdown
<!-- 画像: [説明文] -->
```
- 設備の写真が必要な箇所
- データのグラフが必要な箇所
- メニューやスケジュールの画像が必要な箇所

### 文字数ガイドライン

- **目標範囲**: 1,000-2,700文字
- **最小**: 1,000文字（読み応えを確保）
- **推奨**: 1,500-2,200文字（読みやすさと情報量のバランス）
- **最大**: 2,700文字（長すぎると読まれにくい）

### 表現パターン

**課題を述べる際**
- NG: 「失敗しました」
- OK: 「〜という課題がありました」「〜については真摯に受け止め」

**変更を伝える際**
- 変更内容を明確に
- 背景・理由を説明
- 利用者への影響を考慮した表現

**感謝を表す際**
- 「いつもご利用いただき、ありがとうございます」
- 「おかげさまで〜」
- 「支えられていることを実感」

**締めくくり**
- 必須フレーズ: 「引き続き、Tatami Worksをよろしくお願いいたします！」
- バリエーション可: 「引き続きTatami Worksをどうぞよろしくお願いいたします！」

## トピックカテゴリ別の表現例

### 1. 運営変更
- 受付時間の変更
- 予約制度の導入・変更
- 営業時間の変更
- プラン体系の変更
- ルールの整備

**表現のポイント**:
- 変更の背景を丁寧に説明
- 利用者への影響を明確に
- 今後の柔軟な対応姿勢を示す

**例文**:
```
これまで「会話の声が気になる」「集中したいけど周りが気になってしまう」というお声をいただくこともありました。コワーキングスペースでは、打ち合わせをしたい方と、黙々と作業に集中したい方が同じ空間にいることが多く、どちらのニーズにも応えることが難しい場面がありました。

エリアを明確に分けることで、ミーティングや相談は気兼ねなくできる場所で、集中作業は静かな環境で、それぞれの利用シーンに合った環境を提供できるようになります。
```

### 2. 設備改善
- 家具の新調・追加
- WiFi環境の強化
- 冷蔵庫などの設備追加
- モニター・デバイスの増設

**表現のポイント**:
- 具体的な改善内容（速度、数量など）
- 利用者の作業効率への貢献を強調
- 快適性の向上をアピール

**例文**:
```
11月、ついにWifiの高速化を実現しました！

速度はなんと1.4Gbpsが出るようになりました。これまでの速度と比べると、体感でもかなり違いを感じていただけると思います。ルーターも新しいものに交換したため、接続の安定性も大幅に向上しています。

コワーキングスペースにとって、ネット環境は命とも言える部分です。リモートワークでの大容量ファイルのやり取りや、オンライン会議、動画のアップロードなど、これまで以上に快適にご利用いただけると思います。
```

### 3. サービス追加
- 飲み物メニューの追加
- 特別価格の設定
- 新プランの導入

**表現のポイント**:
- 価格を明示
- 利用条件を明確に
- お得感を伝える

**例文**:
```
5月から、アイスコーヒーの提供を開始しました！価格は**150円**です。

暑くなってくる季節、冷たいコーヒーで一息つきながら作業できる環境を整えました。ドリップコーヒーと同じく、気軽にご利用いただけます。
```

### 4. イベント・コミュニティ
- 勉強会の開催
- もくもく会
- ワークショップ
- 企画者募集

**表現のポイント**:
- コミュニティ形成の意義
- 参加しやすい雰囲気
- 今後の展開への期待

**例文**:
```
Tatami Worksでは、利用者同士の交流やコミュニティ形成も大切にしています。6月から、イベント企画者を募集することにしました。

「こんな勉強会をやってみたい」「このテーマで話し合いたい」といったアイデアがあれば、ぜひお声がけください。場所の提供や告知のサポートなど、できる限りのお手伝いをさせていただきます。
```

## 注意事項

1. **重複チェック**: 必ず既存ファイルを確認し、同じ月の記事を上書きしない
2. **トピック数**: 2-4個を推奨（1個では薄い、5個以上は長すぎる）
3. **目次の一致**: 目次項目とセクション見出しを必ず一致させる
4. **締めくくり**: 標準フレーズを必ず含める
5. **絵文字**: 冒頭のみ使用、本文では過度に使わない
6. **画像**: 実際の画像は追加できないため、コメントでプレースホルダーを示す
7. **ファイル名**: 全角スペースと正確な日本語形式を使用
8. **文字数**: 文字数を確認し、目標範囲（1,000-2,700文字）内に収める
9. **質問は丁寧に**: 各トピックを3段階で質問し、質が高い記事を作成する

## エラーハンドリング

- 同じ月の記事が既に存在する場合: ユーザーに確認し、上書きするか別名で保存するか選択させる
- トピックが0個または5個以上: 警告を出し、推奨範囲に収めるよう提案
- 文字数が1,000文字未満: 内容を充実させるよう提案
- 文字数が3,000文字超: 簡潔にまとめるよう提案
