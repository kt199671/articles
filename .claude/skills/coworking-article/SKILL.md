# コワーキングの場づくり研究室 - note記事作成スキル

「コワーキングの場づくり研究室」マガジン向けの学術的なnote記事を作成するスキルです。

## いつこのスキルを使うか

以下のような依頼があった場合に自動的に発動してください：

- 「コワーキングの記事を書いて」
- 「コワーキング研究室の新しい記事を作成」
- 「note記事を作成」（コワーキング関連の文脈で）
- 「論文記事を書いて」（コワーキング関連の文脈で）

## 記事の保存先

```
/home/kentaro/my-project/articles/note/magazine/コワーキングの場づくり研究室/
```

## 作成フロー

### Phase 1: リサーチ（必須）

1. **既存記事の確認**
   - 同フォルダ内の `.md` ファイルをGlobで検索
   - 各記事のテーマを把握してテーマ重複を避ける

2. **最新情報の収集**
   - WebSearchで学術研究・市場動向・統計データを収集
   - 検索クエリ例：
     - `"コワーキングスペース 研究 2024 2025"`
     - `"coworking space academic research [テーマ] 2024"`

3. **テーマ候補の提案**
   - 既存記事と重複しない3-4個のテーマを提案
   - AskUserQuestionで選択してもらう

### Phase 2: 計画

1. EnterPlanModeで記事構成を設計
2. 計画ファイルに以下を記載：
   - テーマと背景
   - 章立て（8章程度）
   - 参照する学術資源
   - 使用する統計データ

### Phase 3: 執筆

1. TodoWriteで各章の進捗を管理
2. 目標文字数：8,000〜10,000字
3. 各章を順番に執筆
4. 参考文献を末尾に記載

### Phase 4: 仕上げ

1. 文字数確認（`wc -m`）
2. 画像生成用の英語プロンプトを4-5個出力
   - サムネイル用 x 1
   - 挿入画像用 x 3-4

## スタイルガイド

### トーン
- **論文的**：学術的な引用と根拠を重視
- **記事的**：読みやすく、具体例を豊富に
- **客観的**：エッセイっぽさは抑える

### 構成パターン

```markdown
# タイトル：サブタイトル

## 1. はじめに
- 問いかけ、背景説明
- 本稿の目的

## 2-6. 本論（複数章）
- 理論的基盤
- 詳細分析
- 事例・データ

## 7. 実践的示唆
- 運営者向け
- 利用者向け

## 8. 結論

## 参考文献
```

### フォーマット規則
- 見出しは `#` から `####` まで階層的に使用
- **太字** で重要概念を強調
- 統計データには出典を明記
- 専門用語には説明やカッコ書きを付ける
- 章の区切りには `---` を使用
